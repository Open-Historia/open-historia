Open Historia / AI Setup Guide

Connect an AI Backend to Open Historia

Unlock AI-powered diplomacy, dynamically generated historical events, and a strategic advisor by connecting a local large language model (LLM) to your game.

What AI Powers in Open Historia

Ai Diplomacy — negotiate with any nation via natural language chat
Ai Intelligence Briefings — get summaries of any country's status
Ai-Generated Events — dynamic events shaped by your decisions and the world state
Ai Advisor — consult for strategic guidance, economic analysis, and war planning
Ai Combat Resolution — battles resolved by the AI based on troop strength, terrain, and strategy

How It Works

Open Historia communicates with any OpenAI-compatible chat completions API. This means any LLM backend that speaks the standard /v1/chat/completions protocol will work — including popular local LLM inference engines like llama.cpp, Ollama, and LM Studio.

The game's Express server includes a /api/ai/relay endpoint that proxies AI requests to your chosen backend. This keeps API keys on your server and lets the client remain key-free.

Option 1: Ollama (Recommended for Easiest Setup)

Ollama is the simplest way to run LLMs locally. It works on Windows, macOS, and Linux.

Install Ollama

Download from ollama.com and install.

Pull a model

ollama pull llama3.2 — or any model you prefer. Recommended models: llama3.2 (lightweight), mistral (balanced), qwen2.5 (strong reasoning).

Verify the API

Ollama serves at http://localhost:11434/v1 by default. The OpenAI-compatible endpoint is built-in.

Configure in Open Historia

In the game's settings, set the AI provider URL to http://localhost:11434/v1. No API key needed for local use.

Option 2: LM Studio

LM Studio provides a graphical interface for downloading and running models. Great for Windows and macOS users who prefer a GUI.

Install LM Studio

Download from lmstudio.ai.

Download a model

Use the built-in model browser to search and download a model (e.g., Llama 3.2, Mistral 7B, Phi-3).

Start the local server

Go to the "Local Server" tab, load your model, and click "Start Server." The default URL is http://localhost:1234/v1.

Configure in Open Historia

Set the AI provider URL to http://localhost:1234/v1. No API key needed.

Option 3: llama.cpp

llama.cpp is the most performant option for technical users. It runs GGUF-format models with minimal overhead.

Build llama.cpp server

Follow the build instructions for your platform.

Download a GGUF model

Get a GGUF model from Hugging Face (e.g., Llama-3.2-3B-Instruct-Q4_K_M.gguf).

Start the server

./llama-server -m model.gguf --port 8080

Configure in Open Historia

Set the AI provider URL to http://localhost:8080/v1.

Option 4: Cloud / OpenAI-Compatible APIs

You can also use any cloud-based OpenAI-compatible API, including:

  • OpenAI API: https://api.openai.com/v1 — requires an API key
  • Groq: https://api.groq.com/openai/v1 — fast inference with free tier
  • Together.ai: https://api.together.xyz/v1 — open-source models as a service
  • Any OpenAI-compatible proxy: OpenRouter, DeepInfra, Fireworks, etc.

Set the provider URL in the game settings and add your API key if required. The server's relay endpoint forwards the key securely.

Model Recommendations

Model Size Best For Min RAM
Llama 3.2 3B ~3B params Diplomacy chat, advisor 4 GB
Mistral 7B ~7B params Events, diplomacy, advisor 8 GB
Qwen 2.5 7B ~7B params Strong reasoning, all features 8 GB
Phi-3 Mini ~3.8B params Lightweight, fast 4 GB

Smaller models respond faster and use less RAM — ideal for diplomacy chat. Larger models generate richer events and better strategic advice.

Troubleshooting

AI features are greyed out or unresponsive

Make sure your LLM backend is running and accessible. Check that the provider URL in settings matches the backend's actual URL. The game checks for connectivity on load — if the backend isn't reachable, AI features won't activate.

AI responses are slow

Try a smaller model (3B parameters) or enable GPU acceleration in your LLM backend. Ollama and LM Studio both support GPU offloading.

CORS errors when using a remote backend

If connecting to a remote API (not localhost), use the game's relay endpoint at /api/ai/relay — it handles CORS transparently.
