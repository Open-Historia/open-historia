Get Started with Open Historia

Download, install, and launch the free open-source grand strategy game in minutes — on Windows, macOS, Linux, or Android.

Quick Start

Open Historia ships with one-click launchers that handle everything — no manual npm install, no map data downloads, no build commands. Just download and run.

Windows

Double-click Launch Open Historia.bat. The script checks Node.js, downloads map data, installs dependencies, builds the client, and opens the game in your browser at http://localhost:3000.

To update later, run Update Open Historia.bat — it fetches the latest version while preserving your saves, scenarios, and map data.

macOS

Double-click Launch Open Historia.command. On first run, right-click the file and select Open to bypass Gatekeeper. Everything else is automatic.

To update later, run Update Open Historia.command.

Linux

Open a terminal and run:

./"Launch Open Historia.sh"

To update later: ./"Update Open Historia.sh"

Android (Thin Client APK)

Easiest Method

Download pax-historia.apk from the Android release page and open it to install. When Android prompts, allow installs from your browser.

Requirements: Termux must be installed on your Android device. On first launch, the app automatically finds the Termux server on the same phone. To connect to another machine, type its address once — it's remembered between sessions.

The APK is a thin client: the game itself runs on the server you connect to.

Manual Setup

If you prefer to set up manually for development or advanced use:

Step 1 — Clone the repository

git clone https://github.com/Open-Historia/open-historia.git

Step 2 — Enter the project directory

cd open-historia

Step 3 — Download map data

node scripts/fetch-map-assets.mjs

This downloads the large map binaries from GitHub Releases (not included in the repo).

Step 4 — Install dependencies

npm install

Step 5 — Build the client

npm run build

Step 6 — Start the server

node server/server.js

Step 7 — Play

Open http://localhost:3000 in your browser.

Prerequisites

  • Git — to clone the repository
  • Node.js (v18+) — to run the server and build the client

The launcher scripts check for Node.js automatically. If you're doing a manual setup, make sure both are installed first.
